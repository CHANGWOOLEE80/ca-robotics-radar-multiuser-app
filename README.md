# CA Robotics Radar

## 1. Purpose

CA Robotics Radar is a multi-user web application prototype for managing construction automation and robotics technologies.

The purpose of this application is to help the Advanced Construction Technology Group collect, compare, evaluate, and follow up automation and robotics technology items in a structured way.

This repository is intended to be used as a starting point for building AI Agent-based work tools for EPC construction productivity improvement.

## 2. Background

The competitiveness of our group will depend on how effectively we can use AI Agents to improve each member's work process.

Instead of using AI only for simple search or document drafting, we aim to create practical work tools and agents that can support recurring tasks such as:

- Technology radar management
- Vendor and solution comparison
- Field applicability review
- ROI and productivity impact review
- Risk and readiness assessment
- Action item follow-up
- Management reporting support

## 3. Main Functions

The current application concept includes the following functions:

- Register automation and robotics technology items
- Manage technology category and application area
- Review field applicability
- Compare risk, readiness, and productivity impact
- Track follow-up actions
- Support multi-user collaboration
- Provide a structured basis for management review

## 4. Target Users

The target users are:

- Advanced Construction Technology Group members
- Construction automation and robotics reviewers
- Project execution support members
- Quality and productivity improvement staff
- Management reviewers

## 5. Recommended Use Cases

This application can be used for:

1. Managing automation and robotics technology candidates
2. Organizing vendor information
3. Reviewing field deployment potential
4. Comparing technology readiness and application risk
5. Preparing weekly meeting discussion items
6. Tracking action items and responsible persons
7. Building future AI Agent-based work processes

## 6. Implementation Direction

This application should be managed through GitHub as the source repository.

However, because the application is expected to operate as a multi-user server-based app, it should not be treated as a simple GitHub Pages static website.

Recommended implementation approach:

1. Use GitHub for source code management
2. Use Issues for improvement task tracking
3. Use README for project purpose and operation guide
4. Use a server environment such as local PC, internal VM, Docker, or cloud server for execution
5. Use sample data only until security and operation policies are confirmed

## 7. Security Notes

Do not upload confidential company information to this repository.

Do not include:

- Real vendor quotations
- Project-specific confidential data
- Client information
- Internal approval documents
- Personal information
- Passwords, API keys, or access tokens

Use sample data only for the initial prototype.

## 8. Initial Development Steps

The initial implementation should proceed in the following order:

1. Create GitHub repository
2. Add README
3. Upload source files from the ZIP package
4. Review file structure
5. Run the application locally
6. Verify main functions
7. Create improvement Issues
8. Decide internal deployment method
